Отчёт по лабораторной работе
Сравнение алгоритмов сортировки

1. Результаты
Были протестированы 5 алгоритмов сортировки на массивах разного размера.
sorted() — самый быстрый на всех размерах
Merge Sort — быстрый, но требует дополнительную память
Heap Sort — стабильный, но медленнее sorted()
Bubble Sort и Insertion Sort — слишком медленные, на больших массивах не работают (timeout)
2. Анализ
Во сколько раз sorted() быстрее Heap Sort?
На массиве из 1 000 000 элементов:
sorted() быстрее примерно в 23 раза
Почему sorted() такой быстрый?
Написан на языке C, а не на Python
Использует алгоритм Timsort
Хорошо работает с памятью и процессором
Когда лучше использовать Heap Sort?
Если важно экономить память
Если нужна гарантированная сложность O(n log n)
Подходит для систем с ограниченными ресурсами

3. Выводы

В Python всегда лучше использовать sorted() или .sort()
Heap Sort полезен, если память ограничена
Merge Sort быстрый, но «ест» больше памяти
Bubble и Insertion Sort не подходят для больших данных
